## 🛠 Установка и настройка Ollama

Перед использованием ассистента необходимо установить и настроить сервер Ollama.

### 1. Установка Ollama
Выполните следующую команду в терминале (для Linux):
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### 2. Настройка доступа по сети и автозагрузки
По умолчанию Ollama доступна только локально (`127.0.0.1`). Чтобы скрипт мог работать с любого ПК в сети, а служба запускалась автоматически:

1. Создайте файл переопределения сервиса:
   ```bash
   sudo systemctl edit ollama.service
   ```

2. Добавьте следующие строки в открывшийся редактор:
   ```ini
   [Service]
   Environment="OLLAMA_HOST=0.0.0.0:11434"
   ```
   *(Это заставит Ollama слушать все сетевые интерфейсы)*

3. Перезагрузите конфигурацию и запустите службу:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable --now ollama
   ```

4. Проверьте статус:
   ```bash
   systemctl status ollama
   ```
   Если видите `active (running)`, всё готово.

### 3. Установка рекомендуемых моделей
Для работы ассистента нужны модели. Рекомендую установить следующие (выберите в зависимости от мощности вашего ПК):

*   **Для генерации команд (быстрая и точная):**
    ```bash
    ollama pull qwen2.5-coder:1.5b
    # или для более мощных ПК:
    ollama pull qwen2.5-coder:7b
    ```

*   **Для обычных вопросов и чата:**
    ```bash
    ollama pull llama3.2:3b
    ```

Проверить список установленных моделей можно командой: `ollama list`.
